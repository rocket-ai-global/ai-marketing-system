Nguyên tắc: mọi template **dựng sẵn 80% trên ngách Spa** (bản "Sen Spa" điền mẫu đầy đủ), học viên chỉ thay 20% dữ liệu doanh nghiệp mình. Build MỘT LẦN, dùng mọi cohort; nhân bản sang nha khoa/clinic ~90% copy-paste (xem [[Khách Hàng Mục Tiêu — AI Marketing & Sale OS]]).

**Trạng thái:** ⬜ chưa build · 🟨 đang build · ✅ xong (kèm link file khi có).

## Thứ tự build (gợi ý sprint)

1. **Sprint 1 (trước tuyển sinh):** T41-T44 (vận hành) + T08 (Knowledge Base structure) — vì tuyển sinh và Ngày 0 cần trước.
2. **Sprint 2 (trước khai giảng):** toàn bộ 🅿️0 còn lại của Tuần 1-2 (T01-T10, T19) + T22, T24, T27, T28, T30, T33, T37 — đây là "bộ xương sống cài chạy ngay".
3. **Sprint 3 (song song cohort chạy):** 🅿️1 theo tuần, luôn xong trước tuần tương ứng ≥3 ngày.

> Cấu trúc chi tiết từng template (các trường, ví dụ Sen Spa điền mẫu) nằm trong **phần 7 của giáo án ngày tương ứng** — file này chỉ là bản kê + tiến độ. Khi build xong template nào, đổi trạng thái và gắn link file/RocketAgent vào bảng.
